## 3. Architecture

```
PC (home Wi‑Fi)
  https://192.168.18.150:443
        ↓
    leo-proxy (Docker nginx, self-signed TLS)
        ↓
    react-app :80  (static web + /api proxy)
        ↓
    leo-api :8080
        ↓
    postgres :5432 (internal only)

Phone (Tailscale)
  http://100.126.222.96:80
        ↓
    leo-proxy (Docker nginx, plain HTTP)
        ↓
    (same chain as above)
```

### Why HTTP on Tailscale IP?

- Avoids self-signed certificate errors on mobile browsers and React Native.
- Traffic is still encrypted by Tailscale’s wireguard tunnel between devices.
- Simpler than juggling Tailscale Serve, MagicDNS, and Docker nginx on the same ports.

### Why HTTPS on LAN only?

- Local PCs can accept a one-time self-signed cert warning.
- Keeps `http://192.168.18.150` redirecting to HTTPS for LAN users.

## 7. API environment (`apps/api/.env`)

Production API reads env from `apps/api/.env` (not committed with secrets in git — file lives on server).

| Variable | Purpose |
|----------|---------|
| `DATABASE_URL` | PostgreSQL connection (`postgres:5432/leoos` inside Docker) |
| `SESSION_SECRET` | Signs session cookies |
| `SUPERUSER_EMAIL` / `SUPERUSER_PASSWORD` | Bootstrap superuser on first start |
| `NODE_ENV` | `production` |
| `PORT` | `8080` |
| `CORS_ORIGIN` | `https://192.168.18.150,http://100.126.222.96` |
| `COOKIE_SECURE` | `true` — cookies use `secure: "auto"` so HTTP Tailscale + HTTPS LAN both work |
| `OPENAI_API_KEY` | Optional; can also be set in Settings → Passport OCR |

**CORS:** Both LAN (HTTPS) and Tailscale (HTTP) origins must be listed.

**Sessions:** `apps/leo-os/apps/api/src/lib/session.ts` uses `secure: "auto"` when `COOKIE_SECURE=true`, so cookies work over HTTP on the tailnet IP and HTTPS on LAN.

## 11. Tailscale notes

- **Tailscale Serve:** Explicitly **disabled** (`tailscale serve reset` in `go-live.sh`). Not compatible with the chosen “HTTP on `100.x`” model without extra complexity.
- **MagicDNS** (`server.snowy-pride.ts.net`): Available on tailnet but **not used** for Leo OS access.
- **Phone requirement:** Tailscale app must be connected before opening `http://100.126.222.96`.
- **Known tailnet health warning:** “Tailscale can't reach the configured DNS servers” — affects upstream DNS on server, not peer-to-peer `100.x` access.

### SSL troubleshooting history (resolved)

| Symptom | Cause | Resolution |
|---------|-------|------------|
| `ERR_SSL_PROTOCOL_ERROR` on `https://100.126.222.96` | Tailscale Serve + Docker nginx both on 443; cert mismatch | Use HTTP on tailnet IP; disable Serve |
| `ERR_FAILED` on `https://server.snowy-pride.ts.net` | Serve pointed to `127.0.0.1:80` (nothing listening) or cert mismatch | Abandoned MagicDNS URL; use `http://100.126.222.96` |
| LAN works, Tailscale broken | Port binding / wrong Serve backend | Current split: LAN `443`, Tailscale `80` only |

## 14. Application overview (condensed)

**Leo OS** is an ERP + AI OCR platform for a Maldives recruitment/manpower agency.

| Surface | Tech | Self-hosted access |
|---------|------|-------------------|
| Web dashboard | React + Vite + shadcn/ui | `https://192.168.18.150/` |
| Mobile (LEO Admin) | Expo / React Native | API: `http://100.126.222.96` |
| API | Express 5, port 8080 | `/api/*` via react-app nginx |
| Database | PostgreSQL 17 | Docker `postgres`, DB `leoos` |

**Core features:** passport OCR (GPT-4o Vision), companies/clients, billing, LOA, salary, expenses, tasks, RBAC (superuser → admin → client → company → employee → agent).

**Auth:** Cookie sessions (web), Bearer session ID (mobile). Superuser seeded from env on first boot.

**Codegen:** OpenAPI spec drives React Query hooks and Zod schemas (`pnpm --filter @leo/api-spec run codegen` if spec changes).

## 15. Security notes

- Self-signed TLS on LAN only — acceptable for private network; do not expose port 443 to the public internet without a real certificate.
- Tailscale HTTP is acceptable because the tailnet is the trust boundary; do not expose `100.126.222.96:80` outside Tailscale.
- Secrets live in `apps/api/.env` — never commit to git.
- Postgres is not published on host ports.
- `COOKIE_SECURE=true` with `secure: "auto"` sends secure cookies only over HTTPS (LAN); HTTP Tailscale mobile uses Bearer tokens instead.
