# Deployment

Production lives under `/home/adhuhaam/apps`. Source builds from `leo-os/`; runtime is Docker Compose.

**LAN URL:** `https://192.168.18.150/`  
**Tailscale URL:** `http://100.126.222.96/`

## Prerequisites

- Docker + Compose
- External Docker network `homelab`
- pnpm on host (for web builds)
- `apps/api/.env` and `apps/postgresql/.env` filled in
- Ports 80/443 free on the LAN IP (CasaOS stopped via `go-live.sh`)

## One-shot go live

```bash
sudo bash /home/adhuhaam/apps/scripts/go-live.sh
```

Stops CasaOS services, ensures LAN TLS cert exists, recreates proxy/react/api, disables Tailscale Serve, prints URLs and curls health.

## Database

`postgres` service; data in `postgresql/data/`.

On first API start, `bootstrap()` seeds superuser, default permissions, expense categories, and applies safe `ALTER TABLE IF NOT EXISTS` patches.

Manual schema push (dev / intentional):

```bash
cd /home/adhuhaam/apps/leo-os
# DATABASE_URL pointed at reachable Postgres
pnpm --filter @leo/db run push
```

## Deploy API

```bash
cd /home/adhuhaam/apps
docker compose build leo-api-dotnet
docker compose up -d --force-recreate leo-api-dotnet
docker logs leo-api-dotnet --tail 50
# Expect: LEO ASP.NET API listening (PORT=8080)
```

Dockerfile: `leo-os-dotnet/Dockerfile`.
## Deploy web

```bash
cd /home/adhuhaam/apps/leo-os
pnpm deploy:web
```

Runs `@leo/web` build (includes PWA asset generation) and rsyncs `apps/web/dist/` → `/home/adhuhaam/apps/react/app/`. No container restart required for static updates.

Users may need a refresh / reinstall for service worker updates.

## Proxy / TLS

Change nginx or certs → recreate proxy:

```bash
cd /home/adhuhaam/apps
docker compose up -d --force-recreate leo-proxy
```

Certs: `infra/certs/cert.pem` + `key.pem` (SAN: `IP:192.168.18.150`). Generated by `go-live.sh` if missing.

## Rollback

- **Web:** restore previous `react/app/` contents from git or backup
- **API:** rebuild previous `leo-api-dotnet` image from git (see [MIGRATION-DOTNET.md](MIGRATION-DOTNET.md) for historical Express restore via git checkout)

## Post-deploy checklist

1. `docker logs leo-api-dotnet` shows listening
2. Health OK on LAN and Tailscale
3. Login sets session; `/api/auth/me` works
4. Branding endpoint is lightweight (logos separate)
5. Dashboard, master list, companies, expenses, billing load
6. OCR upload → auto LOA
7. Master list shows job titles; salary import labels include job title
8. Optional: PWA install prompt
